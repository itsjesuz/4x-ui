# 4X-UI (پنل مدیریت اکانت Xray) به همراه ربات تلگرام NetFly

پروژه **4X-UI** نسخه اختصاصی و توسعه‌یافته پنل محبوب مدیریت Xray-core به همراه ربات تلگرامی پیشرفته فروش و مدیریت اشتراک **NetFly** است.

این پروژه بر پایه نسخه قدرتمند [3X-UI](https://github.com/MHSanaei/3x-ui) اثر توسعه‌دهنده گرامی **MHSanaei** پیاده‌سازی شده است. صمیمانه از تلاش‌های ارزشمند ایشان تشکر می‌کنیم و از کاربران دعوت می‌کنیم برای حمایت از این توسعه‌دهنده، از [صفحه گیت‌هاب MHSanaei](https://github.com/MHSanaei) بازدید و حمایت کنند.

---

## ۱. پنل مدیریت 4X-UI

این پنل امکان راه‌اندازی و مدیریت کانکشن‌های لینوکسی مبتنی بر هسته Xray را با ویژگی‌های پیشرفته فراهم می‌کند.

### نصب و راه‌اندازی پنل
برای نصب نسخه جدید یا ارتقای نسخه فعلی پنل روی سرور لینوکس خود، دستور زیر را اجرا کنید:

```bash
bash <(curl -Ls https://raw.githubusercontent.com/itsjesuz/4x-ui/main/install.sh)
```

## ۲. ربات تلگرام NetFly (پوشه `bot/`)

ربات تلگرام **NetFly** یک ربات پرسرعت، منعطف و غنی برای فروش، تحویل خودکار و مدیریت اشتراک‌های VPN است که با فریم‌ورک‌های مدرن پایتون (`aiogram 3.x` و `httpx`) پیاده‌سازی شده و به صورت مستقیم به API پنل‌های X-UI متصل می‌شود.

کد منبع ربات در پوشه `bot/` این مخزن قرار گرفته است.

### ویژگی‌های کلیدی ربات:
- **تحویل و تمدید خودکار**: تخصیص فوری کلاینت و تحویل لینک‌های سابسکریپشن و کانفیگ‌ها پس از تایید رسید توسط ادمین.
- **تمدید هوشمند**: تمدید حجم و زمان اشتراک‌های فعال بدون تغییر کانفیگ یا UUID کاربر.
- **مدیریت اشتراک توسط کاربر**: مشاهده حجم باقی‌مانده و مصرفی به صورت زنده، تغییر کلید (UUID) برای لغو دسترسی‌های غیرمجاز، و فعال/غیرفعال کردن موقت کانکشن.
- **مدیریت چندسروره (Multi-Node)**: پشتیبانی از اتصال چندین سرور X-UI و تخصیص هوشمند لوکیشن‌ها.
- **سیستم نقش‌ها**: سیستم کنترل دسترسی پیشرفته با سطوح دسترسی مختلف برای مالک اصلی، مدیران، پشتیبان‌ها و ناظرین.
- **قفل کانال (Join Gate)**: اجبار کاربران به عضویت در کانال‌های اسپانسر قبل از استفاده از ربات.

### راهنمای نصب و راه‌اندازی ربات
برای اجرای ربات، پایتون نسخه ۳.۱۰ یا بالاتر مورد نیاز است.

۱. وارد پوشه ربات شوید:
```bash
cd bot
```

۲. پکیج‌های پایتون را نصب کنید:
```bash
pip install -r requirements.txt
```

۳. فایل نمونه تنظیمات `.env.example` را کپی کرده و نام آن را به `.env` تغییر دهید:
```bash
cp .env.example .env
```

۴. فایل `.env` را ویرایش کرده و اطلاعات زیر را وارد کنید:
- `BOT_TOKEN`: توکن دریافتی از `@BotFather`
- `ADMIN_IDS`: شناسه‌های عددی تلگرام ادمین‌های ربات
- `DB_PATH`: مسیر دیتابیس ربات (مانند `netfly.db`)
- `REQUIRED_CHANNEL_ID`: آیدی عددی کانال قفل عضویت تلگرام (همراه با `-100`)

۵. ربات را اجرا کنید:
```bash
python bot.py
```

## ساختار دیتابیس ربات (SQLite)

ربات برای ذخیره‌سازی اطلاعات از یک پایگاه داده SQLite سبک و سریع استفاده می‌کند که شامل جداول اصلی زیر است:

### ۱. جدول `users` (کاربران)
اطلاعات تلگرام تمامی کاربرانی که ربات را استارت زده‌اند.
- `user_id` (کلید اصلی): شناسه عددی تلگرام کاربر.
- `username` ، `first_name` ، `last_name` ، `lang_code`
- `is_banned`: وضعیت مسدودیت کاربر در ربات (0 یا 1).

### ۲. جدول `locations` (لوکیشن‌ها / سرورها)
اطلاعات پنل‌های X-UI متصل به ربات.
- `base_url`: آدرس کامل پنل X-UI (شامل پورت و مسیر مخفی).
- `api_token`: توکن Bearer اتصال به پنل.
- `inbound_ids`: شناسه‌های مجاز inbound در پنل (فرمت JSON).
- `price_base` ، `price_per_gb` ، `price_per_day`: قیمت‌گذاری اختصاصی سرور.

### ۳. جدول `orders` (سفارش‌ها و اشتراک‌ها)
نگهداری اطلاعات خریدها، وضعیت‌های پرداخت و مشخصات کلاینت ساخته شده در پنل.
- `status`: وضعیت سفارش (`awaiting_payment` ، `awaiting_review` ، `approved` ، `declined` ، `provisioned` ، `failed` ، `completed_renewal`).
- `screenshot_file_id`: آیدی عکس فیش واریزی ارسال شده کاربر در سرورهای تلگرام.
- `xui_email`: ایمیل ثبت‌شده کلاینت در پنل (شناسه یکتا مانند `nf101`).
- `xui_client_uuid`: شناسه UUID کانکشن کاربر.
- `sub_links`: لینک‌های کانفیگ استخراج‌شده (فرمت JSON).
- `renew_of_order_id`: شناسه سفارش اصلی برای ارجاع تمدیدها.

### ۴. جدول `settings` (تنظیمات سراسری)
ذخیره تعرفه‌های مالی سراسری ربات، شماره کارت بانکی، وضعیت اشتراک تست رایگان و کانال اسپانسر.
