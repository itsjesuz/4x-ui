پروژه **4X-UI** نسخه اختصاصی و توسعه‌یافته پنل مدیریت قدرتمند Xray به همراه ربات پیشرفته تلگرامی فروش و مدیریت اشتراک **NetFly** است. این ابزار به شما امکان می‌دهد خدمات VPN خود را کاملاً هوشمند، امن و خودکار به مشتریان عرضه کنید.

> [!NOTE]
> این پروژه بر پایه نسخه قدرتمند [3X-UI](https://github.com/MHSanaei/3x-ui) اثر توسعه‌دهنده گرامی **MHSanaei** پیاده‌سازی شده است. صمیمانه از تلاش‌های ارزشمند ایشان تشکر می‌کنیم و از کاربران دعوت می‌کنیم برای حمایت از این توسعه‌دهنده، از [صفحه گیت‌هاب MHSanaei](https://github.com/MHSanaei) بازدید و حمایت کنند.

## ⚡ ویژگی‌های برجسته پروژه

### ۱. پنل مدیریت 4X-UI
- **هسته قدرتمند Xray-core**: پشتیبانی از آخرین پروتکل‌ها (VLESS, VMess, Trojan, ShadowSocks) و کانفیگ‌های پیشرفته (Reality, TLS, gRPC, WebSocket).
- **مدیریت پهنای باند و زمان**: محاسبه زنده و دقیق مصرف ترافیک و قطع خودکار اتصال در صورت اتمام اشتراک.
- **نصب آسان و سریع**: اسکریپت نصب یکپارچه با قابلیت مدیریت دیتابیس و سرویس‌ها.

### ۲. ربات تلگرامی NetFly (در پوشه `bot/`)
- 🤖 **تحویل و تمدید کاملاً خودکار**: تخصیص فوری کلاینت و ساخت اشتراک در پنل، و تحویل لینک‌های سابسکریپشن پس از تایید فیش توسط ادمین.
- ⏳ **تمدید هوشمند بدون تغییر کانکشن**: تمدید حجم و زمان اشتراک‌های فعال بدون نیاز به تغییر UUID یا آدرس کانفیگ کاربران.
- 👤 **پنل کاربری مستقل**: مشاهده لحظه‌ای ترافیک مصرفی و زمان باقی‌مانده، تغییر UUID برای لغو دسترسی‌های غیرمجاز، و دریافت فایل کانفیگ یا بارکد QR.
- 🌐 **پشتیبانی از Multi-Node (چند سرور)**: اتصال همزمان چندین پنل X-UI به ربات و تخصیص هوشمند کلاینت‌ها بر اساس موقعیت انتخابی کاربر.
- 🛡️ **سیستم سطوح دسترسی (RBAC)**: کنترل دسترسی ادمین‌ها با نقش‌های مالک اصلی، مدیر، پشتیبان و ناظر.
- 📢 **قفل کانال (Join Gate)**: اجبار کاربران به عضویت در کانال‌های تلگرام تعریف‌شده قبل از دسترسی به عملکردهای ربات.

## 🔧 نصب و راه‌اندازی پنل 4X-UI

برای نصب یا ارتقای نسخه پنل روی سرور لینوکس خود (ترجیحاً Ubuntu 20.04 یا بالاتر)، دستور زیر را در ترمینال اجرا کنید:

```bash
bash <(curl -Ls https://raw.githubusercontent.com/itsjesuz/4x-ui/main/install.sh)
```

## 🤖 راه‌اندازی ربات تلگرامی NetFly

ربات تلگرامی NetFly به صورت خودکار در زمان نصب پنل توسط اسکریپت نصب یکپارچه (`install.sh`) راه‌اندازی و به عنوان یک سرویس سیستمی لینوکس تنظیم می‌شود. در حین اجرای اسکریپت نصب، مقادیر پیکربندی زیر از شما پرسیده شده و به طور خودکار در فایل تنظیمات ذخیره خواهند شد:

| نام متغیر | توضیحات | نمونه مقدار |
| :--- | :--- | :--- |
| `BOT_TOKEN` | توکن دریافتی ربات تلگرام از BotFather | `12345678:AA...` |
| `ADMIN_IDS` | شناسه عددی تلگرام ادمین‌های اصلی (با ویرگول جدا شود) | `1111111,2222222` |
| `DB_PATH` | مسیر ذخیره‌سازی دیتابیس SQLite ربات | `netfly.db` |
| `REQUIRED_CHANNEL_ID` | آیدی عددی کانال قفل عضویت (با پیشوند `100-`) | `-100123456789` |

> [!TIP]
> پس از نصب خودکار، در صورت نیاز به تغییر هر یک از این مقادیر، می‌توانید فایل تنظیمات واقع در مسیر `/usr/local/4x-ui/bot/.env` را ویرایش کرده و سپس سرویس ربات را با دستور زیر راه‌اندازی مجدد کنید:
> ```bash
> systemctl restart netfly-bot
> ```

## 🗄️ ساختار پایگاه داده ربات (SQLite Schema)

ربات NetFly از یک پایگاه داده SQLite برای مدیریت اطلاعات استفاده می‌کند. ساختار جداول اصلی به شرح زیر است:

### ۱. جدول `users` (کاربران)
نگهداری مشخصات کاربرانی که ربات را استارت زده‌اند.

| نام ستون | نوع داده | توضیحات |
| :--- | :--- | :--- |
| `user_id` 🔑 | INTEGER | شناسه عددی تلگرام کاربر (کلید اصلی) |
| `username` | TEXT | نام کاربری تلگرام |
| `first_name` | TEXT | نام کوچک کاربر |
| `last_name` | TEXT | نام خانوادگی کاربر |
| `lang_code` | TEXT | زبان کلاینت تلگرام |
| `is_banned` | INTEGER | وضعیت مسدودیت (0 = فعال، 1 = مسدود) |
| `created_at` | TEXT | تاریخ و زمان اولین تعامل با ربات |

### ۲. جدول `locations` (لوکیشن‌ها / سرورها)
اطلاعات سرورها و پنل‌های متصل به ربات.

| نام ستون | نوع داده | توضیحات |
| :--- | :--- | :--- |
| `id` 🔑 | INTEGER | شناسه عددی لوکیشن (کلید اصلی) |
| `name` | TEXT | نام نمایشی لوکیشن (مانند `Germany 🇩🇪`) |
| `base_url` | TEXT | آدرس اتصال به API پنل X-UI |
| `api_token` | TEXT | توکن اهراز هویت API پنل |
| `inbound_ids` | TEXT | آرایه شناسه‌های inbound مجاز (فرمت JSON) |
| `enabled` | INTEGER | وضعیت فعال بودن کلی لوکیشن (0 یا 1) |
| `sub_url_template` | TEXT | الگوی ساخت لینک اشتراک (شامل `{subId}`) |
| `price_base` | INTEGER | هزینه پایه ساخت اشتراک |
| `price_per_gb` | INTEGER | هزینه ترافیک مصرفی به ازای هر گیگابایت |
| `price_per_day` | INTEGER | هزینه دوره زمانی اشتراک به ازای هر روز |
| `is_test` | INTEGER | آیا لوکیشن مخصوص اشتراک تست رایگان است؟ (0 یا 1) |
| `purchase_enabled` | INTEGER | امکان خرید مستقیم از این لوکیشن توسط کاربر (0 یا 1) |

### ۳. جدول `orders` (سفارش‌ها و اشتراک‌ها)
ذخیره اطلاعات خریدهای ثبت شده، کانفیگ‌های کاربران و وضعیت‌های تایید مالی.

| نام ستون | نوع داده | توضیحات |
| :--- | :--- | :--- |
| `id` 🔑 | INTEGER | شناسه عددی سفارش (کلید اصلی) |
| `user_id` | INTEGER | شناسه عددی خریدار (ارجاع به `users.user_id`) |
| `location_id` | INTEGER | شناسه لوکیشن (ارجاع به `locations.id`) |
| `volume_gb` | INTEGER | حجم اشتراک خریداری شده به گیگابایت |
| `duration_days` | INTEGER | مدت زمان اشتراک به روز |
| `price` | INTEGER | مبلغ پرداخت شده به ریال / تومان |
| `status` | TEXT | وضعیت سفارش (`awaiting_payment`, `approved`, `provisioned`, `declined`, ...) |
| `screenshot_file_id` | TEXT | شناسه تصویر فیش واریزی در تلگرام |
| `xui_email` | TEXT | ایمیل ثبت شده کلاینت در پنل X-UI (مانند `nf11`) |
| `xui_client_uuid` | TEXT | شناسه UUID کانکشن کلاینت در پنل |
| `sub_links` | TEXT | لینک‌های کانفیگ استخراج‌شده (فرمت JSON) |
| `created_at` | TEXT | تاریخ ثبت سفارش |
| `renew_of_order_id` | INTEGER | ارجاع به شناسه سفارش اصلی (برای پیگیری تمدیدها) |

### ۴. جدول `service_packages` (پلن‌های پیش‌فرض)
پکیج‌های تعریف‌شده توسط مدیریت برای خرید دستی آسان.

| نام ستون | نوع داده | توضیحات |
| :--- | :--- | :--- |
| `id` 🔑 | INTEGER | شناسه پکیج (کلید اصلی) |
| `location_id` | INTEGER | شناسه لوکیشن مربوطه |
| `volume_gb` | INTEGER | حجم پلن به گیگابایت |
| `duration_days` | INTEGER | مدت پلن به روز |
| `price` | INTEGER | قیمت نهایی پلن |
| `enabled` | INTEGER | وضعیت فعال بودن پلن (0 یا 1) |
